Quick Start (Docker)

Source code: github.com/grussdorian/cjp-decentralised

git clone https://github.com/grussdorian/cjp-decentralised
cd cjp-decentralised
docker compose up -d

That's it. Your node will pin the site to IPFS and announce itself to the global mirror registry every 15 minutes. No domain or public IP needed — but if you have one, read below.

Serve with your own domain (optional)

If you have a VPS or server with a public hostname, you can serve the site as a clearweb fallback. Set the MIRROR_URL environment variable to your public URL and the daemon will broadcast it in every heartbeat — your domain will appear automatically in the Active mirrors list below, and in the GitHub registry, for anyone to visit. 

# In your docker-compose.override.yml (or edit docker-compose.yml):
services:
  mirror:
    environment:
      MIRROR_URL: "https://cjp.yourdomain.example"
      COUNTRY: "IN"   # your ISO country code

Then add a reverse proxy (nginx/Caddy/Traefik) to forward your domain to the bundled nginx on http://localhost:80, which serves the auto-updating mirror content. The verification badge at the bottom of every page will automatically confirm whether your mirror is serving authentic, signed content. 

# Minimal nginx snippet
server {
    server_name cjp.yourdomain.example;
    location / {
        proxy_pass http://127.0.0.1:8080;
        add_header Cache-Control "no-cache, must-revalidate";
    }
}

How the mirror registry works

Every mirror daemon broadcasts a signed Nostr heartbeat event every 15 minutes tagged #cjp-mirrors. The heartbeat includes the pinned CID, country, and — if set — the mirror's public URL. The active mirrors list below is built live from these events, queried directly from public Nostr relays in your browser. No central server. No registration. No list to maintain. If your node is alive and connected to a Nostr relay, it is counted.

Mirrors with a public URL are automatically linked from this page and from the GitHub repository. Visitors can click any listed mirror and verify its badge independently.

Manual Setup (no Docker)

1. Install IPFS

# Download kubo (go-ipfs) from https://dist.ipfs.tech
ipfs init
ipfs daemon &

2. Pin the site

# Get the latest CID from latest.json in the repo
ipfs pin add <CID>

3. Run the mirror daemon

MIRROR_URL=https://cjp.yourdomain.example \
COUNTRY=IN \
./mirror --ipfs http://localhost:5001

How content updates work

The party publishes a signed latest.json file whenever the site is updated. The mirror daemon fetches it every 15 minutes, verifies the Ed25519 signature against a list of trusted party public keys embedded in the binary, and pins the new CID if the signature is valid. A tampered or unsigned update is silently rejected.
